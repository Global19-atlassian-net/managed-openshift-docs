= Understanding private connections for AWS
:context: osd-understanding-aws-private


[role="_abstract"]
Amazon Web Services (AWS) permits link:https://access.redhat.com/node/3610411[Customer Portal Organization Administrators] and cluster owners to enable AWS Identity and Access Management (IAM) users to have federated access to the AWS Management Console for their {product-title} cluster. This AWS access is necessary if the cluster privacy is set to *Private*. You cannot access your private cluster until you configure private connections in AWS.

.Prerequisites

- An AWS user and account has been created.
- The AWS user has access to the OpenShift Dedicated AWS account.
